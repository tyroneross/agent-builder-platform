# Document Structure Skill

## Purpose

Produce Word documents that are easier to scan, review, and reuse.

## Inputs

- Document goal.
- Audience.
- Claims or schedule data.
- Required decisions, risks, and appendices.

## Outputs

- Structured `.docx`.
- Markdown mirror.
- Tables for claims, owners, schedule blocks, or risks.
- Appendix notes.

## Procedure

1. Start with the conclusion.
2. Separate decision, evidence, risks, and next actions.
3. Use tables when data has repeated fields.
4. Keep claims and uncertainty labels visible.
5. Add appendices only when they support review or reuse.

# Accuracy Review

## Verified

- Narrow tool surfaces reduce local-model failure risk.
- Persistent lessons should require evaluation before promotion.
- Large multi-agent systems fail through bad verification and poor inter-agent communication.

## Inferred

- Agent handoffs should include task goal, available context, constraints, expected output, and stop conditions.
- Research decks should be generated from a claim table, not from prose alone.
- Local models should use shorter loops and stronger termination criteria than frontier cloud models.

## Needs Review

- A Chief of Staff agent can coordinate specialist agents if it owns acceptance criteria and final verification.

## Rule

- Unsupported claims do not enter decks or Word documents without a visible label.
